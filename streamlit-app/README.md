# RUL Ops Copilot — Streamlit Demo

An interactive demo showcasing the **uncertainty-aware agentic framework** for turbofan engine Remaining Useful Life (RUL) prediction.

## What It Does

The Copilot runs a **4-step pipeline** for each engine:

| Step | Tool | Description |
|------|------|-------------|
| 1 | `predict_rul` | CNN-Transformer + MC Dropout → RUL mean, std, 95% CI |
| 2 | `explain_prediction` | Integrated Gradients → top contributing sensors |
| 3 | `retrieve_knowledge` | BM25 search over curated fault-tree & glossary |
| 4 | `recommend_action` | Grounded recommendation with abstention logic |

## Pages

- **Overview** — Project summary and model details
- **Engine Analysis** — Run the full Copilot pipeline on a single engine (with live uncertainty, explanation, KB retrieval, and recommendations)  
- **Fleet Dashboard** — Batch inference on all 100 test engines with fleet-level statistics  
- **About** — Architecture details and safety guardrails  

## Prerequisites

From the project root (`RUN_2/`):

```bash
pip install -r requirements.txt
```

Make sure you have:
- A trained CNN-Transformer checkpoint at `checkpoints/cnn_transformer_FD001_best.pt`  
  (generated by running weeks 1–6 or `src/train.py`)
- The C-MAPSS dataset files in `data/`
- The knowledge base files in `kb/`

## How to Run

```bash
cd RUN_2
streamlit run streamlit-app/app.py
```

The app will open in your browser at `http://localhost:8501`.

### Quick Start

1. Navigate to **🔍 Engine Analysis**
2. Select an engine from the dropdown (shows true RUL for reference)
3. Adjust MC Dropout samples (100 default) and escalation threshold
4. Click **🚀 Run Copilot Analysis**
5. View the full report: prediction, confidence interval, sensor attribution, KB context, recommendation, and safety checks

### Fleet Analysis

1. Navigate to **📊 Fleet Dashboard**
2. Set MC samples (lower = faster, 30 recommended for fleet)
3. Click **⚡ Analyze Entire Fleet**
4. View scatter plots, distributions, and a sortable engine table

## Notes

- The model runs on CPU by default (inference is lightweight)
- MC Dropout with 100 samples takes ~1–2 seconds per engine
- Fleet analysis of all 100 engines takes ~30–60 seconds
- Escalation threshold controls when the Copilot abstains from automated recommendations
